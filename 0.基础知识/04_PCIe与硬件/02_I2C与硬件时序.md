# I2C 与硬件时序

I2C 适合低速外设通信，常见设备包括 EEPROM、传感器、时钟芯片、电源管理芯片等。

## 基本信号

| 信号 | 含义 |
| --- | --- |
| SCL | 时钟线，由主设备控制时钟 |
| SDA | 数据线，用来传输地址、数据和 ACK |
| 上拉电阻 | I2C 总线通常依赖上拉电阻保持高电平 |

常见速率：

| 模式 | 典型速率 |
| --- | --- |
| Standard mode | 100 kbit/s |
| Fast mode | 400 kbit/s |

## 起始与停止条件

正常数据传输时，SDA 应该在 SCL 低电平期间变化，在 SCL 高电平期间保持稳定。例外是起始和停止条件。

| 条件 | 判定方式 |
| --- | --- |
| Start | SCL 为高电平时，SDA 从高到低 |
| Stop | SCL 为高电平时，SDA 从低到高 |

## 关键时序

| 参数 | 英文 | 含义 | 关注边沿 |
| --- | --- | --- | --- |
| `tSU:STA` | Start Setup Time | 起始条件前，SCL 需要保持高电平的最短时间 | SDA 下降沿前 |
| `tHD:STA` | Start Hold Time | 起始条件后，SCL/SDA 需要保持的最短时间 | SDA 下降沿后 |
| `tSU:DAT` | Data Setup Time | SCL 上升沿前，数据需要稳定的最短时间 | SCL 上升沿前 |
| `tHD:DAT` | Data Hold Time | SCL 下降沿后，数据需要继续保持的最短时间 | SCL 下降沿后 |
| `tSU:STO` | Stop Setup Time | 停止条件前，SDA 变化需要满足的建立时间 | SDA 上升沿前 |

## 为什么要关注保持时间

从设备通常在 SCL 上升沿采样 SDA。如果主设备在 SCL 下降沿附近立刻改变 SDA，信号反射、耦合或总线电容可能影响刚完成的采样，导致边界时序不稳定。

`tHD:DAT` 的意义是给从设备采样电路一个缓冲时间，让数据位稳定结束，再允许 SDA 变化。

```text
情况 A：保持时间足够

SDA   _______██████████____________
                  |       |
SCL   ------------|_______|--------
              上升沿采样  下降沿后等待再改变 SDA

情况 B：保持时间太小

SDA   _______█████_________________
                  ||
SCL   ------------|_|--------------
              上升沿采样  SDA 很快变化，可能影响稳定性
```

## i2c-tools 常用命令

| 命令 | 用途 |
| --- | --- |
| `i2cdetect -l` | 查看当前系统有哪些 I2C 总线 |
| `i2cdetect -F 0` | 查看 0 号总线支持的功能 |
| `i2cdetect -r -y 0` | 扫描 0 号总线上的设备 |
| `i2cset -f -y 1 0x20 0x77 0x3f` | 向 i2c-1 上 `0x20` 设备的 `0x77` 寄存器写 `0x3f` |
| `i2cget -f -y 1 0x20 0x77` | 读取 i2c-1 上 `0x20` 设备的 `0x77` 寄存器 |
| `i2cdump -f -y -a 1 0x38` | 读取 i2c-1 上 `0x38` 设备的寄存器内容 |
| `ls /sys/bus/i2c/devices/` | 查看系统上的 I2C 总线及设备地址 |

## 注意点

- `-f` 会强制访问设备，可能绕过内核驱动保护，调试时要谨慎。
- `-y` 表示不再交互确认，脚本中常用，但手动操作时要确认总线号和地址。
- 写寄存器前要确认器件地址、寄存器偏移和数据位宽，避免误改硬件状态。
