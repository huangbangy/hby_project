# C 语言知识：SysV 信号量

信号量常用于进程间同步。下面示例是 SysV 信号量的加锁和解锁写法。

## 核心概念

| 概念 | 含义 |
| --- | --- |
| P 操作 | `sem_op = -1`，申请资源，信号量减 1。常用于加锁 |
| V 操作 | `sem_op = 1`，释放资源，信号量加 1。常用于解锁 |
| `semget` | 根据 key 获取信号量 ID |
| `semop` | 对信号量执行 P/V 操作 |
| `SEM_UNDO` | 进程异常退出时，内核尽量回滚该进程对信号量造成的影响 |

## 加锁操作

```c
int32_t sem_lock(key_t key)
{
    int32_t sem_id;
    struct sembuf sem_b;

    sem_id = semget((key_t)key, 1, 0666);
    if (sem_id == -1) {
        syslog(LOG_ERR, "semget failed for key %x, error: %m", key);
        return ENV_ERROR;
    }

    sem_b.sem_num = 0;
    sem_b.sem_op = -1;
    sem_b.sem_flg = SEM_UNDO;

    if (semop(sem_id, &sem_b, 1) == -1) {
        syslog(LOG_ERR, "semop failed for key %x, error: %m", key);
        return ENV_ERROR;
    }

    return ENV_OK;
}
```

## 解锁操作

```c
int32_t sem_unlock(key_t key)
{
    int32_t sem_id;
    struct sembuf sem_b;

    sem_id = semget((key_t)key, 1, 0666);
    if (sem_id == -1) {
        syslog(LOG_ERR, "semget failed for key %x, error: %m", key);
        return ENV_ERROR;
    }

    sem_b.sem_num = 0;
    sem_b.sem_op = 1;
    sem_b.sem_flg = SEM_UNDO;

    if (semop(sem_id, &sem_b, 1) == -1) {
        syslog(LOG_ERR, "semop failed for key %x, error: %m", key);
        return ENV_ERROR;
    }

    return ENV_OK;
}
```

## 注意点

- 如果信号量还没有创建，获取时通常需要 `IPC_CREAT`，例如 `semget(key, 1, 0666 | IPC_CREAT)`。
- `ENV_OK`、`ENV_ERROR` 是项目里的自定义返回值，实际代码需要确保它们已经定义。
- 加锁成功后要保证异常路径也能解锁，否则容易造成其他进程一直等待。
- 共享内存本身没有锁机制，经常需要搭配信号量保护读写。
