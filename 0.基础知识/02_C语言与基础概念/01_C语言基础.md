# C 语言基础与常见错误

## 常用函数

| 函数/写法 | 作用 | 备注 |
| --- | --- | --- |
| `access(path, F_OK)` | 判断文件或目录是否存在 | 成功返回 `0`，失败返回 `-1` |
| `access(path, R_OK/W_OK/X_OK)` | 判断是否可读、可写、可执行 | 常用于启动前检查 |
| `strstr(str1, str2)` | 判断 `str2` 是否是 `str1` 的子串 | 找到返回首次出现地址，否则返回 `NULL` |
| `strtoul(tmp, NULL, 10)` | 按 10 进制把字符串转成无符号长整型 | 第三个参数是进制 |
| `itoa(16, str, 8)` | 把整数转成指定进制字符串 | `itoa` 不是标准 C，跨平台时谨慎使用 |
| `BIT(3)` | 常见位宏，通常等价于 `(1 << 3)` | 表示第 3 位置 1 |

## 编译、链接与可执行文件

```bash
gcc -c main.c
```

`-c` 表示只编译成目标文件，不进行链接。如果要生成可执行文件，需要把目标文件链接起来。

C 程序变成可执行文件的大致流程：

1. 预处理：展开头文件、删除注释、替换宏。
2. 编译：把 C 代码翻译成汇编代码。
3. 汇编：把汇编代码转成目标文件。
4. 链接：合并目标文件和库，生成可执行文件。

## 编码与数据表示

- GBK 编码中，一个汉字通常占 2 字节。
- UTF-8 是变长编码，常见中文通常占 3 字节。
- 正数在计算机中通常按原码理解，负数实际以补码形式存储。
- 无符号整数打印常用 `%u`。

## `static` 与 `extern`

| 关键字 | 使用位置 | 作用 |
| --- | --- | --- |
| `static` | 文件作用域变量/函数 | 限制在当前 `.c` 文件可见，其他文件不能用 `extern` 访问 |
| `static` | 函数内部变量 | 存放在静态存储区，函数结束后值仍保留 |
| `extern` | 全局变量声明 | 声明变量在别的地方定义，本文件可以引用 |
| `extern` | 函数声明 | 显式声明函数全局可见，函数默认通常就是 `extern` |

```c
static int count;
extern g_para_t envpara;
```

## `register` 与 `volatile`

- `register`：请求编译器尽量把变量放到 CPU 寄存器中，只是建议，不是强制。
- `volatile`：告诉编译器每次都要真实访问该内存地址，不要把访问优化掉。常见于硬件寄存器、中断共享变量等场景。

## 内存与 `sizeof`

```c
int arr[4] = {1, 2, 3, 2};
int len_arr = sizeof(arr);   // 通常是 16，4 个 int，每个 4 字节

char *p = malloc(12);
printf("%zu\n", sizeof(p));  // 64 位系统通常是 8，sizeof 指针本身
```

`new/delete` 是 C++ 运算符，`malloc/free` 是 C 标准库函数。C 代码中申请和释放内存要成对出现。

## 常见内存问题

| 问题 | 说明 |
| --- | --- |
| 空指针使用 | 指针为 `NULL` 时仍解引用 |
| 野指针 | 内存释放后，指针仍保存原地址并继续使用 |
| 内存越界 | 写超数组或申请空间范围 |
| 内存未释放 | `malloc` 后没有 `free` |
| 申请和释放机制不一致 | 例如 C++ 中 `new` 搭配 `free` |
| 重复释放 | 同一块内存 `free` 多次 |

结构体字段最好初始化。未初始化字段可能包含栈上的随机残留值，使用后会导致不可预测行为。

```c
struct demo d = {0};
```

## 进程间通信

| 方式 | 适合场景 |
| --- | --- |
| 管道 | 同一台机器上的简单进程通信 |
| 消息队列 | 同一台机器上的进程间消息传递 |
| 共享内存 | 高性能共享数据，需要额外加锁 |
| 信号量 | 进程间同步，经常保护共享内存 |
| Socket | 同机或跨服务器通信 |

## 互斥锁与自旋锁

| 锁 | 等待方式 | 适合场景 |
| --- | --- | --- |
| 互斥锁 mutex | 获取不到锁时睡眠，让出 CPU | 进程上下文、等待时间可能较长 |
| 自旋锁 spinlock | 获取不到锁时忙等，不睡眠 | 中断上下文或非常短的临界区 |

持有自旋锁时不能睡眠，否则可能造成死锁。互斥锁可以睡眠，所以不能在中断上下文中使用。
