为什么是野指针
	内存释放后：free(p) 释放了内存，但指针 p 仍然持有该内存的地址。
	野指针行为：在释放内存后，p 指向的内存可能被操作系统回收或重新分配给其他程序。因此，向该地址写入数据会导致未定义行为（undefined behavior），包括程序崩溃或数据损坏。
3：内存越界
4：内存未释放
5:内存生成和释放机制不一致
6：内存空间重复释放

volatile 的核心作用就是告诉编译器"不要优化这个内存访问，老老实实地每次都去硬件那里读取！

"睡眠"（sleep） = 进程主动让出CPU，等待某个条件满足后被唤醒

持有自旋锁 = 必须快速完成，不能让出CPU

自旋锁：为了在中断上下文也能用，设计成不能睡眠（因为中断根本无法睡眠）
互斥锁：为了在进程上下文能高效处理长等待，设计成可睡眠（避免浪费CPU）

当一个线程试图获取已经被其他线程持有的 mutex 时，它会被挂起。挂起意味着该线程会被操作系统调度器切换出去，直到锁可用时再被唤醒。

当一个线程试图获取已被持有的自旋锁时，它会进行忙等待（自旋），不断检查锁是否可用，而不是进入休眠状态。
