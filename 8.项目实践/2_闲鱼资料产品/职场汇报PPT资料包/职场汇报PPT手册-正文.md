## 1. 汇报前先想清楚三件事

很多汇报写不好，不是因为不会排版，而是因为没有先想清楚对象、目的和结论。

你可以先回答三个问题：

- 这次汇报给谁看：直属领导、跨部门同事、客户、团队成员。
- 对方最关心什么：结果、风险、资源、下一步、经验沉淀。
- 汇报后希望对方做什么：知道进展、确认方向、给资源、做决策。

不要一上来就打开 PPT。先写一段 100 字的汇报摘要：

```text
这次汇报主要说明【项目/工作】的当前进展。
核心结论是：【一句话结论】。
目前最大的风险是：【风险】。
下一步需要：【动作/资源/决策】。
```

只要这段话写清楚，后面的 PPT 就不会散。

## 2. 周报月报的三段式结构

周报月报最怕写成流水账。建议使用“目标 - 进展 - 计划”的结构。

### 2.1 周报结构

```text
一、本周目标
- 本周围绕【目标】推进。

二、本周完成
- 完成【事项】，结果是【结果】。
- 推进【事项】，当前进度【百分比/阶段】。

三、问题与下周计划
- 问题：【问题】，影响：【影响】。
- 下周计划：【动作】。
- 需要协同：【对象+事项】。
```

### 2.2 月报结构

月报比周报更强调结果和趋势：

- 本月重点目标。
- 关键成果和数据变化。
- 重要项目进度。
- 问题、原因和改进。
- 下月重点计划。

如果没有漂亮数据，也不要编。可以写“完成了什么基础动作”“发现了什么问题”“下月准备怎么改”。

## 3. 述职汇报怎么写贡献

述职不是把所有工作堆上去，而是把贡献讲成主线。

推荐结构：

1. 岗位职责：我负责什么。
2. 年度重点：我今年主要解决什么问题。
3. 关键动作：我做了哪些动作。
4. 结果反馈：产生了什么结果。
5. 经验复盘：我学到了什么。
6. 下一阶段计划：接下来怎么做。

贡献表达可以使用这个公式：

```text
在【背景/问题】下，我负责【职责】。
通过【动作/方法】，推动了【过程变化】。
最终带来【结果/反馈】。
后续准备从【方向】继续优化。
```

注意：没有数据时，可以写“定性结果”，但不要编造数字。

## 4. 项目复盘怎么讲问题

项目复盘不是甩锅，也不是只讲成绩。好的复盘要把“发生了什么、为什么发生、下次怎么改”说清楚。

项目复盘模板：

| 模块 | 要写什么 |
|---|---|
| 背景 | 为什么做这个项目 |
| 目标 | 原计划解决什么问题 |
| 动作 | 关键推进步骤 |
| 结果 | 完成情况、数据或现象 |
| 问题 | 过程中的卡点 |
| 原因 | 人、流程、资源、判断上的原因 |
| 改进 | 下次具体怎么做 |

问题写法建议：

- 不写“大家配合不好”，改成“需求确认节点不清，导致返工”。
- 不写“时间不够”，改成“前期评估遗漏了素材审核时间”。
- 不写“效果一般”，改成“转化环节缺少明确引导，后续需要优化页面结构”。

## 5. PPT 页面标题公式

PPT 页面标题不要只写“背景介绍”“数据分析”。最好写成结论句。

常用公式：

- 事实 + 结论：本月线索量增长，但有效转化不足。
- 问题 + 影响：素材审核周期偏长，影响活动上线节奏。
- 动作 + 结果：通过流程前置，交付周期缩短。
- 对比 + 判断：A 方案成本低，但长期维护压力更大。

页面结构可以这样拆：

```text
第 1 页：一句话结论
第 2 页：背景和目标
第 3 页：关键动作
第 4 页：结果和数据
第 5 页：问题和原因
第 6 页：下一步计划
```

## 6. 数据页和结论页写法

数据页不要堆图表，要回答“这组数据说明了什么”。

数据页模板：

```text
数据对象：【指标】
观察周期：【时间】
主要变化：【变化】
可能原因：【原因】
对业务影响：【影响】
下一步动作：【动作】
```

结论页建议只保留 3 个重点：

- 一个核心判断。
- 两到三个证据。
- 一个下一步动作。

## 7. 演讲稿和过渡句模板

很多人 PPT 做完，讲的时候会卡。可以提前写每页 3 句话：

```text
这一页主要说明【主题】。
从数据/过程上看，【关键内容】。
所以我的判断/建议是【结论】。
```

常用过渡句：

- 前面讲完背景，接下来我们看具体进展。
- 结果之外，更重要的是过程中暴露出的三个问题。
- 基于这些问题，下一步我计划做三件事。
- 这部分需要大家重点关注的是资源和时间节点。

## 8. 汇报前自检清单

发布或汇报前，建议检查：

- 有没有一句话结论。
- 每页标题是不是结论句。
- 数据是否真实，来源是否清楚。
- 有没有把问题和改进写具体。
- 需要领导决策的事项是否明确。
- 是否删除了和主题无关的页面。
- 演讲稿是否能在规定时间内讲完。

汇报的重点不是把自己显得很忙，而是让对方更快理解现状、风险和下一步。
