## 1. 这份资料适合谁

这份资料适合三类人：

- 应届生：经历不多，不知道怎么把课程、社团、实习和项目写清楚。
- 转行者：原经历和目标岗位不完全匹配，需要重新组织表达。
- 职场新人：工作做了不少，但简历写成流水账，面试时讲不出重点。

它不适合想伪造经历、包装学历、购买内推或寻找保过服务的人。简历优化的底线是：让真实经历更清楚、更匹配，而不是把没有的东西写成有。

## 2. 求职前先做 3 件事

### 2.1 明确目标岗位

不要用一份简历投所有岗位。至少先选 1 到 3 个目标岗位，比如：

- 新媒体运营
- 产品助理
- 数据分析助理
- 行政人事
- Java 开发实习
- 嵌入式软件工程师

目标越清楚，简历越容易写。

### 2.2 收集 10 个岗位 JD

打开招聘软件，找到目标岗位，收集 10 条 JD。重点看：

- 岗位职责里反复出现的动词。
- 任职要求里反复出现的能力。
- 工具、证书、学历、行业经验要求。
- 哪些是必须项，哪些是加分项。

### 2.3 建立经历素材库

先不要急着写简历，先把经历列出来：

| 类型 | 可写内容 |
|---|---|
| 实习经历 | 做过什么、负责什么、结果如何 |
| 项目经历 | 背景、目标、分工、方法、成果 |
| 校园经历 | 社团、比赛、志愿活动、课程作业 |
| 工作经历 | 职责、流程、工具、数据、改进 |
| 技能证书 | 软件、语言、证书、作品链接 |

## 3. 简历的基本结构

一份适合普通求职者的简历，建议使用以下结构：

1. 基本信息
2. 求职意向
3. 教育背景
4. 实习/工作经历
5. 项目经历
6. 技能证书
7. 作品或补充信息

### 基本信息

保留必要信息即可：

- 姓名
- 手机
- 邮箱
- 城市
- 求职意向
- 作品链接或 GitHub 链接

不建议放太多无关信息，比如身份证号、详细家庭住址、无关兴趣爱好。

### 求职意向

写清楚目标岗位和城市，例如：

```text
求职意向：产品助理 / 广州 / 可尽快到岗
```

### 教育背景

应届生可以放靠前，工作多年后可以放靠后。写法：

```text
2022.09-2026.06  XX 大学  计算机科学与技术  本科
相关课程：数据结构、操作系统、计算机网络、数据库
```

## 4. 经历描述的 STAR 公式

STAR 是面试和简历都能用的表达框架：

- S：Situation，背景是什么。
- T：Task，任务是什么。
- A：Action，你做了什么。
- R：Result，结果如何。

简历里不需要把 S、T、A、R 都写出来，但要体现动作和结果。

差的写法：

```text
负责公众号运营，写文章，做排版。
```

更好的写法：

```text
负责公众号内容运营，完成选题策划、文章撰写与排版发布；结合后台阅读数据调整标题和发布时间，保持每周 3 篇稳定更新。【可补充数据：阅读量、涨粉数、转化数】
```

常用动词：

- 梳理、搭建、优化、整理、推动、协调、复盘、分析、设计、执行。
- 提升、降低、减少、缩短、沉淀、规范、完成、交付。

## 5. 应届生简历怎么写

应届生常见问题是“没有经历”。实际上你可以从这些地方找素材：

- 课程大作业
- 毕设或科研训练
- 实习
- 社团活动
- 比赛
- 兼职
- 志愿活动
- 个人作品

### 课程项目写法

```text
XX 管理系统课程项目 | 项目负责人
- 负责需求拆解、数据库表设计和核心页面开发。
- 使用 XX 技术完成用户登录、数据查询和权限管理功能。
- 项目最终完成课程展示，并沉淀项目文档和演示材料。
```

### 校园经历写法

```text
XX 学院迎新活动 | 宣传组成员
- 参与活动宣传物料制作和线上通知发布，覆盖新生群体约 300 人。
- 协助整理报名信息和现场流程，提升签到效率。
```

如果没有准确数据，不要编。可以写“约”“累计”“覆盖”，但要能解释来源。

## 6. 转行简历怎么写

转行简历的重点不是隐藏过去，而是找共通能力。

常见共通能力：

- 沟通协调
- 数据整理
- 用户理解
- 项目推进
- 内容表达
- 流程优化
- 客户服务

转行表达公式：

```text
过去经历中的可迁移能力 + 目标岗位需要的能力 + 已经补足的技能
```

示例：

```text
过往在门店运营中长期负责客户沟通、销售数据整理和活动执行，具备一线用户理解和执行落地经验。近期系统学习了 Excel 数据分析和基础 SQL，希望转向数据运营助理岗位。
```

## 7. 岗位 JD 匹配方法

把 JD 拆成三类：

| 类别 | 例子 | 简历处理 |
|---|---|---|
| 必备条件 | 学历、年限、软件、语言 | 简历中明确出现 |
| 核心能力 | 数据分析、沟通、项目推进 | 经历中重点体现 |
| 加分项 | 行业经验、作品、证书 | 有就放，没有不硬编 |

JD 里反复出现的词，最好在简历中自然出现。比如岗位要求“用户增长、活动运营、数据复盘”，你的经历里就应该能看到相关表达。

## 8. 面试自我介绍

自我介绍不要背户口本，也不要泛泛说“我认真负责”。建议控制在 1 分钟左右。

结构：

1. 我是谁。
2. 我为什么匹配这个岗位。
3. 我有什么相关经历。
4. 我希望在这个岗位上做什么。

模板：

```text
面试官您好，我叫【姓名】，毕业于【学校/专业】。
我这次应聘的是【岗位】。我认为自己和岗位比较匹配的地方有两点：
第一是【能力一】，我在【经历】中做过【具体事情】；
第二是【能力二】，我曾经【具体成果】。
我也提前了解了贵公司的【业务/岗位方向】，希望有机会在这个岗位上继续积累，并为团队提供稳定的执行和学习能力。
```

## 9. 高频面试题回答框架

### 你为什么想来我们公司？

回答结构：

```text
了解公司业务 + 岗位匹配 + 自己能贡献什么
```

### 你的优势是什么？

不要只说形容词。用经历证明优势。

```text
我的优势是执行和复盘能力。比如在【经历】中，我负责【任务】，当时遇到【问题】，我通过【动作】解决，最后【结果】。
```

### 你的缺点是什么？

不要说致命缺点，也不要说“太追求完美”。建议讲一个真实但可改进的问题。

```text
我之前在任务拆解上不够细，容易先做再想。后来我开始用清单拆分任务，先确认目标、截止时间和交付标准，现在这方面已经改善很多。
```

### 你有什么想问我的？

可以问：

- 这个岗位前三个月最重要的目标是什么？
- 团队目前最希望新人补上的能力是什么？
- 这个岗位的日常工作节奏是怎样的？
- 如果入职，您建议我提前补充哪些知识？

## 10. 求职沟通模板

### 打招呼

```text
您好，我对贵公司的【岗位】很感兴趣。我有【相关经历/技能】，简历已发送，方便的话希望您帮忙看一下是否匹配，谢谢。
```

### 追问进度

```text
您好，想跟进一下我之前投递的【岗位】目前是否有进一步安排。如果暂时不匹配，也希望能收到反馈，方便我后续调整，谢谢。
```

### 约面试时间

```text
您好，我这边可以参加面试的时间是【时间段】。如果团队有其他安排，我也可以配合调整。
```

### 面试后感谢

```text
您好，感谢今天的面试交流。通过沟通我对岗位和团队有了更清楚的了解，也更确认自己对这个方向感兴趣。期待后续反馈，谢谢。
```

## 11. 投递记录与复盘

建议做一个投递表：

| 公司 | 岗位 | 投递日期 | 渠道 | 状态 | 备注 |
|---|---|---|---|---|---|
| XX 公司 | 产品助理 | 7/1 | Boss | 已投递 | 需要改项目经历 |

每周复盘：

- 投了多少份？
- 哪些岗位有回复？
- 哪些简历版本回复率更高？
- 面试被问到最多的问题是什么？
- 下周简历要改哪里？

## 12. 常见误区

### 误区一：一份简历投所有岗位

不同岗位的关键词不同，建议至少准备 2 到 3 个版本。

### 误区二：经历写成职责清单

不要只写“负责”。要写动作、方法和结果。

### 误区三：过度包装

包装不是编造。面试官很容易追问细节，编出来的内容经不起追问。

### 误区四：只准备答案，不准备例子

面试需要具体经历。每个能力点最好准备一个真实案例。

### 误区五：不复盘

求职不是投完就结束。每一次没回复、每一次面试卡住，都是优化材料的线索。

---

## 结语

好的求职资料不是帮你变成另一个人，而是帮你把真实经历讲清楚。求职真正要解决的是三件事：

1. 让简历和岗位匹配。
2. 让经历表达有重点。
3. 让面试回答经得起追问。

只要这三件事做好，求职效率就会明显提升。
